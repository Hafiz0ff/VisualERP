# Роли и права доступа

Ниже описаны базовые роли. В дальнейшем права должны собираться из `Permission`, а не быть жестко зашитыми только в названия ролей.

## Owner / Director

Основные задачи:

- видеть общую картину по запасам, производству и отгрузкам;
- контролировать ключевые показатели;
- просматривать аудит действий.

Разрешенные действия:

- полный просмотр всех модулей;
- просмотр отчетов и истории изменений;
- утверждение чувствительных операций в будущих версиях;
- управление верхнеуровневыми настройками.

## Administrator

Основные задачи:

- администрирование системы;
- настройка ролей, пользователей и модулей;
- поддержка базовых справочников.

Разрешенные действия:

- создание и блокировка пользователей;
- назначение ролей;
- управление настройками модулей;
- изменение справочников и конфигурации профиля отрасли;
- просмотр системного аудита.

## Warehouse Manager

Основные задачи:

- приемка товаров и сырья;
- учет остатков на складе;
- перемещения и корректировки в рамках своих полномочий.

Разрешенные действия:

- создавать приемки;
- вести складские остатки;
- оформлять передачи в цех;
- выполнять инвентаризацию;
- создавать списания и корректировки по регламенту;
- просматривать складские отчеты.

## Workshop Master

Основные задачи:

- запуск и сопровождение производства;
- контроль расхода материалов;
- фиксация выпуска готовой продукции.

Разрешенные действия:

- создавать и обновлять производственные заказы;
- списывать материалы в производство;
- фиксировать выпуск;
- просматривать остатки цеха;
- видеть рецептуры / BOM, необходимые для работы.

## Accountant / Finance Viewer

Основные задачи:

- контролировать стоимость, выручку и финансовые показатели по мере появления финансового модуля.

Разрешенные действия:

- просмотр данных по закупочным ценам;
- просмотр отчетов по себестоимости и выручке;
- выгрузка отчетов;
- отсутствие права на операционное изменение складских и производственных документов по умолчанию.

## Sales / Shipment Manager

Основные задачи:

- организация отгрузок клиентам;
- контроль доступности готовой продукции.

Разрешенные действия:

- создавать и проводить отгрузки;
- просматривать остатки готовой продукции;
- печатать или экспортировать документы отгрузки;
- видеть историю исполнения отгрузок.

## Auditor / Read-only

Основные задачи:

- проверка данных;
- просмотр истории операций без вмешательства.

Разрешенные действия:

- только просмотр модулей, к которым открыт доступ;
- просмотр аудита;
- отсутствие прав на изменение, удаление и проведение документов.

## Общие принципы прав

- Права должны выдаваться по действиям, а не только по экрану.
- Чувствительные операции должны логироваться в `AuditLog`.
- Удаление критичных данных должно быть либо запрещено, либо заменено мягким архивированием и аудитом.
- Отраслевая специфика не должна менять модель ролей, только словарь и настройки.
